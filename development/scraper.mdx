The Realtors' Practice scraper is a Python microservice that collects property listings from Nigerian real estate websites.

## Tech Stack

- **Framework**: FastAPI 0.115.12
- **Server**: Uvicorn 0.34.3
- **Browser Automation**: Playwright 1.52.0
- **HTML Parsing**: BeautifulSoup 4.13.4
- **Data Validation**: Pydantic 2.11.3
- **Task Queue**: Celery 5.4.0
- **Cache**: Redis 5.2.1
- **HTTP Client**: httpx 0.28.1
- **LLM Fallback**: Google Gemini 0.2.2

## Scraping Pipeline

The scraper follows this workflow:

```
1. Receive Job Request
   ↓
2. Load Site Configuration
   ↓
3. Scrape Listing Pages
   ↓
4. Extract Property URLs
   ↓
5. Scrape Property Details
   ↓
6. Parse & Validate Data
   ↓
7. Enrich with LLM (if needed)
   ↓
8. Send to Backend API
   ↓
9. Update Job Status
```

## Playwright vs Requests

### Use Requests (Faster)

- Static HTML sites
- No JavaScript rendering needed
- Fast, lightweight

```python
import httpx
from bs4 import BeautifulSoup

async with httpx.AsyncClient() as client:
    response = await client.get(url)
    soup = BeautifulSoup(response.text, 'lxml')
    # Parse...
```

### Use Playwright (Slower, but powerful)

- JavaScript-heavy sites
- Infinite scroll
- CAPTCHAs or anti-bot measures

```python
from playwright.async_api import async_playwright

async with async_playwright() as p:
    browser = await p.chromium.launch()
    page = await browser.new_page()
    await page.goto(url)
    await page.wait_for_selector('.property-card')
    html = await page.content()
    # Parse...
```

## Data Validation

Use Pydantic models to validate scraped data:

```python
from pydantic import BaseModel, Field, validator
from typing import Optional

class PropertyData(BaseModel):
    title: str
    price: Optional[float] = None
    bedrooms: Optional[int] = None
    bathrooms: Optional[int] = None
    location: Optional[str] = None
    url: str
    
    @validator('price')
    def validate_price(cls, v):
        if v and v < 0:
            raise ValueError('Price must be positive')
        return v

# Usage
data = PropertyData(
    title="3 Bedroom Flat",
    price=5000000,
    url="https://..."
)
```

## LLM Enrichment

For ambiguous or missing data, use Gemini LLM:

```python
from google import genai
import os

client = genai.Client(api_key=os.getenv('GEMINI_API_KEY'))

def enrich_property(raw_data: dict) -> dict:
    prompt = f"""
    Extract structured property data from this text:
    {raw_data['description']}
    
    Return JSON with: bedrooms, bathrooms, property_type, area
    """
    
    response = client.models.generate_content(
        model='gemini-2.0-flash-exp',
        contents=prompt
    )
    
    # Parse response and merge with raw_data
    enriched = {**raw_data, **parse_llm_response(response.text)}
    return enriched
```

## Sending Data to Backend

After scraping and validation, send data to the backend:

```python
import httpx

async def send_to_backend(properties: list[dict]):
    async with httpx.AsyncClient() as client:

            f"{BACKEND_URL}/api/scrape/ingest",
            json=properties,
            headers={"X-Internal-Key": INTERNAL_API_KEY}
```

## Rate Limiting

Respect site rate limits:

```python
import asyncio
from time import time

class RateLimiter:
    def __init__(self, requests_per_second: float = 1.0):
        self.delay = 1.0 / requests_per_second
        self.last_request = 0
    
    async def wait(self):
        now = time()
        time_since_last = now - self.last_request
        if time_since_last < self.delay:
            await asyncio.sleep(self.delay - time_since_last)
        self.last_request = time()

# Usage
limiter = RateLimiter(requests_per_second=2)

for url in urls:
    await limiter.wait()
    await scrape(url)
```

## Job Types

As defined in Prisma schema:

- **PASSIVE_BULK**: Scheduled full-site scrapes
- **ACTIVE_INTENT**: User-triggered search scrapes
- **RESCRAPE**: Re-scrape specific properties
- **SCHEDULED**: Cron-based scrapes
