Realtors' Practice displays properties on interactive maps with clustering, custom markers, and provider switching.

## Map Providers

Users can choose from three map providers (property-map.tsx:27-58):

<CardGroup cols={3}>
  <Card title="OpenStreetMap" icon="map" color="#7ebc6f">
    **Free & default**. Uses MapLibre GL JS with OSM tiles. No API key required.
  </Card>
  <Card title="Mapbox" icon="location-dot" color="#4264fb">
    **Premium styling**. Requires Mapbox API key. Vector tiles with smooth zoom.
  </Card>
  <Card title="Google Maps" icon="earth-americas" color="#4285f4">
    **Familiar UX**. Requires Google Maps API key. Street View integration.
  </Card>
</CardGroup>

### Provider Configuration

Map provider preference is stored in user settings:

```typescript
// hooks/use-map-provider.ts
const { provider } = useMapProvider();
// Returns: "osm" | "mapbox" | "google"
```

<Info>
If Mapbox or Google Maps is selected but API keys are missing, users see a friendly fallback message (property-map.tsx:38-54).
</Info>

## Map Component Architecture

The system uses dynamic imports to avoid loading unused map libraries:

```typescript
// property-map.tsx:9-15
const OpenStreetMap = dynamic(
  () => import("./osm-map").then((mod) => mod.OSMMap),
  {
    ssr: false,
    loading: () => <Skeleton className="w-full h-full min-h-[400px] rounded-xl" />,
  }
);
```

<Tip>
Dynamic imports reduce initial bundle size by ~150KB. Map libraries only load when the map is visible.
</Tip>

## Clustering Algorithm

When properties are close together, markers cluster automatically:

### Cluster Behavior

| Zoom Level | Cluster Radius | Max Properties |
|------------|----------------|----------------|
| 1-8 (country) | 80px | 1000+ |
| 9-12 (state) | 60px | 100-500 |
| 13-15 (city) | 40px | 10-50 |
| 16+ (street) | No clustering | Individual markers |

<Info>
Clusters show the total count in a badge. Clicking a cluster zooms to expand it.
</Info>

## Map Bounds & Auto-Zoom

The map automatically fits all visible properties:

```typescript
// Calculate bounds from property coordinates
const bounds = properties
  .filter(p => p.latitude && p.longitude)
  .reduce((acc, p) => acc.extend([p.longitude!, p.latitude!]), new LngLatBounds());

map.fitBounds(bounds, { padding: 50 });
```

### Zoom Behavior

<Steps>
  <Step title="Initial load">
    Map zooms to fit all search results.
  </Step>
  <Step title="Filter applied">
    Map re-fits to show only filtered properties.
  </Step>
  <Step title="Single property selected">
    Map zooms to level 16 (street view).
  </Step>
</Steps>

## Geocoding Integration

Properties without coordinates are geocoded during scraping (pipeline/enricher.py):

```python
all_properties = await enrich_property(all_properties)
# Adds latitude/longitude via Google Maps Geocoding API
```

### Geocoding Priority

1. **Scraper extracts coordinates**: If the site provides lat/lng, use them directly.
2. **Google Maps Geocoding**: If coordinates are missing, geocode `fullAddress`.
3. **Fallback to area centroid**: If geocoding fails, use the center of the `area` (e.g., "Lekki").

<Warning>
Properties without coordinates don't appear on the map. Aim for 90%+ geocoding coverage.
</Warning>

## Coordinate System

All coordinates use the **WGS 84** standard (EPSG:4326):

```typescript
// schema.prisma:242-243
latitude  Float?
longitude Float?
```

<Info>
Latitude is Y-axis (North/South), Longitude is X-axis (East/West). Lagos is approximately `6.5°N, 3.4°E`.
</Info>

## Performance Optimization

### Marker Clustering

Without clustering, rendering 1000+ markers freezes the browser. Clustering reduces DOM nodes by 90%+.

### Viewport Culling

Only properties within the current map bounds are rendered:

```typescript
const visibleProperties = properties.filter(p => {
  if (!p.latitude || !p.longitude) return false;
  return map.getBounds().contains([p.longitude, p.latitude]);
});
```

### Lazy Loading Tiles

Map tiles load on-demand as users pan/zoom. No preloading of hidden areas.

<Tip>
Use vector tiles (Mapbox/OSM) instead of raster tiles for 60% faster rendering.
</Tip>

## Map Data Schema

Each property's map data comes from these fields (schema.prisma:233-244):

```prisma
model Property {
  // Location text
  fullAddress   String?
  locationText  String?
  estateName    String?
  area          String?
  lga           String?
  state         String? @default("Lagos")
  
  // Coordinates
  latitude      Float?
  longitude     Float?
  landmarks     String[] @default([])
}
```

### Indexed Coordinates

Coordinate queries use a composite index for fast spatial lookups:

```prisma
@@index([latitude, longitude])
```

## Real-Time Updates

When new properties are scraped, the map updates via WebSocket:

```typescript
// hooks/use-socket.ts
socket.on("property:created", (property: Property) => {
  if (property.latitude && property.longitude) {
    addMarkerToMap(property);
  }
});
```

<Info>
New markers animate in with a "drop" effect and pulse 3 times.
</Info>
