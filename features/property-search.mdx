## Search Parameters

### Query String

The search endpoint accepts these parameters (search.controller.ts:14-25):

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `q` | string | `""` | Search query (keywords, locations, descriptions) |
| `limit` | number | `20` | Number of results to return |
| `offset` | number | `0` | Pagination offset |
| `facets` | string[] | `["categoryName", "listingType", "bedrooms", "state"]` | Fields to return faceted counts for |
| `filters` | string[] | `[]` | Filter expressions (e.g., `price < 5000000`) |
| `sort` | string[] | `["createdAt:desc"]` | Sort order |

### Default Facets

```typescript
const facets = facetsParam 
  ? facetsParam.split(",") 
  : ["categoryName", "listingType", "bedrooms", "state"];
```

Facets return aggregated counts for filtering:

```json
{
  "facets": {
    "state": { "Lagos": 1203, "Abuja": 456, "Rivers": 89 },
    "bedrooms": { "2": 340, "3": 567, "4": 298 },
    "listingType": { "SALE": 892, "RENT": 671 }
  }
}
```

## Graceful Degradation

If Meilisearch is unavailable, the search system returns empty results without breaking the UI:

```typescript
// search.controller.ts:40-43
if (error.message && error.message.includes("failed")) {
  return sendSuccess(res, 
    { hits: [], nbHits: 0, facets: {} }, 
    "Search unavailable (Meilisearch down)"
  );
}
```

<Tip>
Users can still browse properties via direct navigation or saved searches when Meilisearch is down.
</Tip>

## Recent Searches

The system stores recent queries in `localStorage` for quick access:

```typescript
// search-bar.tsx:141-145
function saveRecentSearch(query: string) {
  const recent = getRecentSearches().filter((q) => q !== query);
  recent.unshift(query);
  localStorage.setItem(
    RECENT_SEARCHES_KEY, 
    JSON.stringify(recent.slice(0, MAX_RECENT))
  );
}
```

<Info>
Recent searches are client-side only and persist across sessions.
</Info>
