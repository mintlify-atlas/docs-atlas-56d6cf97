## Base URL

The Realtors' Practice API is accessible at:

<CodeGroup>
```text Production
https://realtors-practice-new-api.onrender.com/api
```

```text Development
http://localhost:5000/api
```
</CodeGroup>

## Authentication

All API requests require authentication using Supabase JWT tokens. Include your access token in the `Authorization` header:

```bash
Authorization: Bearer YOUR_ACCESS_TOKEN
```

See the [Authentication](/api/authentication) guide for details on obtaining tokens.

## Response Format

All API endpoints return JSON responses with a standardized structure:

<ResponseField name="success" type="boolean" required>
  Indicates whether the request was successful
</ResponseField>

<ResponseField name="data" type="any">
  The response payload (present on successful requests)
</ResponseField>

<ResponseField name="error" type="string">
  Error message (present on failed requests)
</ResponseField>

<ResponseField name="message" type="string">
  Optional human-readable message
</ResponseField>

### Success Response Example

```json
{
  "success": true,
  "data": {
    "id": "clx123abc",
    "title": "5 Bedroom Duplex in Lekki",
    "price": 85000000
  },
  "message": "Property retrieved successfully"
}
```

### Error Response Example

```json
{
  "success": false,
  "error": "Property not found",
  "message": "No property exists with ID clx123abc"
}
```

## Rate Limiting

API requests are rate-limited to prevent abuse:

| Endpoint Category | Limit | Window |
|------------------|-------|--------|
| General API | 300 requests | 15 minutes |
| Authentication | 10 requests | 1 hour |
| Search | 200 requests | 5 minutes |

<Note>
  Development environments have higher limits. Rate limit headers are included in all responses.
</Note>

Rate limit information is returned in response headers:

```
RateLimit-Limit: 300
RateLimit-Remaining: 299
RateLimit-Reset: 1678901234
```

## Error Codes

The API uses standard HTTP status codes:

| Code | Meaning | Description |
|------|---------|-------------|
| `200` | OK | Request succeeded |
| `201` | Created | Resource created successfully |
| `400` | Bad Request | Invalid request parameters |
| `401` | Unauthorized | Missing or invalid authentication |
| `403` | Forbidden | Insufficient permissions |
| `404` | Not Found | Resource does not exist |
| `409` | Conflict | Resource already exists (duplicate) |
| `422` | Unprocessable Entity | Validation errors |
| `429` | Too Many Requests | Rate limit exceeded |
| `500` | Internal Server Error | Server error |
| `503` | Service Unavailable | Temporary service outage |

## Pagination

List endpoints support cursor-based pagination:

<ParamField query="page" type="number" default="1">
  Page number (1-indexed)
</ParamField>

<ParamField query="limit" type="number" default="20">
  Number of results per page (max: 100)
</ParamField>

Pagination metadata is included in list responses:

```json
{
  "success": true,
  "data": {
    "items": [...],
    "pagination": {
      "page": 1,
      "limit": 20,
      "total": 487,
      "totalPages": 25,
      "hasMore": true
    }
  }
}
```

## Quick Start

Here's a complete example of fetching properties:

<CodeGroup>
```bash cURL
curl -X GET "https://realtors-practice-new-api.onrender.com/api/properties?limit=10" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const response = await fetch(
  'https://realtors-practice-new-api.onrender.com/api/properties?limit=10',
  {
    headers: {
      'Authorization': `Bearer ${accessToken}`,
      'Content-Type': 'application/json'
    }
  }
);

const { success, data } = await response.json();

if (success) {
  console.log('Properties:', data.items);
}
```

```python Python
import requests

response = requests.get(
    'https://realtors-practice-new-api.onrender.com/api/properties',
    params={'limit': 10},
    headers={
        'Authorization': f'Bearer {access_token}',
        'Content-Type': 'application/json'
    }
)

data = response.json()

if data['success']:
    print('Properties:', data['data']['items'])
```
</CodeGroup>

## Real-Time Updates

In addition to REST endpoints, the platform supports real-time updates via Socket.io:

```javascript
import io from 'socket.io-client';

const socket = io('https://realtors-practice-new-api.onrender.com', {
  auth: {
    token: accessToken
  }
});

// Listen for new property matches
socket.on('search:match', (match) => {
  console.log('New property match:', match);
});

// Listen for scrape progress
socket.on('job:progress', (progress) => {
  console.log('Scrape progress:', progress);
});
```

See individual endpoint documentation for available Socket.io events.
