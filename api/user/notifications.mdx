## List Notifications

Retrieve a paginated list of notifications for the authenticated user.

```http
GET /api/notifications
```

### Query Parameters

<ParamField query="limit" type="number" default="20">
  Number of notifications to return per page (max 100)
</ParamField>

<ParamField query="offset" type="number" default="0">
  Number of notifications to skip for pagination
</ParamField>

<ParamField query="read" type="boolean">
  Filter by read status. Omit to return all notifications.
  - `true` - Only read notifications
  - `false` - Only unread notifications
</ParamField>

### Example Request

<CodeGroup>
```bash cURL (All)
curl "https://api.realtorspractice.com/api/notifications?limit=20&offset=0" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```bash cURL (Unread)
curl "https://api.realtorspractice.com/api/notifications?read=false&limit=10" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://api.realtorspractice.com/api/notifications?read=false&limit=20',
  {
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN'
    }
  }
);
const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.realtorspractice.com/api/notifications',
    headers={'Authorization': 'Bearer YOUR_TOKEN'},
    params={'read': 'false', 'limit': 20}
)
data = response.json()
```
</CodeGroup>

### Response

<ResponseField name="success" type="boolean" required>
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="object" required>
  Notification data and metadata
  
  <ResponseField name="notifications" type="array" required>
    Array of notification objects
    
    <ResponseField name="id" type="string" required>
      Notification ID (CUID)
    </ResponseField>
    
    <ResponseField name="type" type="enum" required>
      Notification type: `NEW_MATCH`, `PRICE_DROP`, `SCRAPE_COMPLETE`, `SCRAPE_FAILED`, or `SYSTEM`
    </ResponseField>
    
    <ResponseField name="title" type="string" required>
      Notification title
    </ResponseField>
    
    <ResponseField name="message" type="string" required>
      Notification message body
    </ResponseField>
    
    <ResponseField name="data" type="object">
      Additional notification metadata (structure varies by type)
    </ResponseField>
    
    <ResponseField name="read" type="boolean" required>
      Whether the notification has been read
    </ResponseField>
    
    <ResponseField name="readAt" type="string">
      ISO 8601 timestamp when notification was marked as read
    </ResponseField>
    
    <ResponseField name="createdAt" type="string" required>
      ISO 8601 timestamp when notification was created
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="unreadCount" type="number" required>
    Total number of unread notifications for this user
  </ResponseField>
</ResponseField>

<ResponseField name="pagination" type="object" required>
  Pagination metadata
  
  <ResponseField name="total" type="number" required>
    Total number of notifications matching the filter
  </ResponseField>
  
  <ResponseField name="page" type="number" required>
    Current page number (1-based)
  </ResponseField>
  
  <ResponseField name="limit" type="number" required>
    Items per page
  </ResponseField>
</ResponseField>


<CodeGroup>
```bash cURL

  -H "Authorization: Bearer YOUR_TOKEN"
```

## Mark as Read

Mark a single notification as read.

```http
PATCH /api/notifications/:id/read
```

### Path Parameters

<ParamField path="id" type="string" required>
  The notification ID to mark as read
</ParamField>

### Example Request

<CodeGroup>
```bash cURL
curl -X PATCH https://api.realtorspractice.com/api/notifications/cm5abc123def456789/read \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const notificationId = 'cm5abc123def456789';
const response = await fetch(
  `https://api.realtorspractice.com/api/notifications/${notificationId}/read`,
  {
    method: 'PATCH',
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN'
    }
  }
);
const data = await response.json();
```

```python Python
import requests

notification_id = 'cm5abc123def456789'
response = requests.patch(
    f'https://api.realtorspractice.com/api/notifications/{notification_id}/read',
    headers={'Authorization': 'Bearer YOUR_TOKEN'}
)
data = response.json()
```
</CodeGroup>

### Response

## Notification Types

Notifications are categorized into five types, each with a specific data structure:

### NEW_MATCH

Triggered when properties matching a saved search are found.

**Data structure:**
```json
{
  "savedSearchId": "string",
  "savedSearchName": "string",
  "matchCount": 3,
  "propertyIds": ["string"]
}
```

### PRICE_DROP

Triggered when a tracked property's price decreases.

**Data structure:**
```json
{
  "propertyId": "string",
  "propertyTitle": "string",
  "oldPrice": 150000000,
  "newPrice": 145000000,
  "difference": -5000000,
  "percentageChange": -3.33
}
```

### SCRAPE_COMPLETE

Triggered when a user-initiated scrape job completes successfully.

**Data structure:**
```json
{
  "jobId": "string",
  "totalListings": 87,
  "newListings": 12,
  "updatedListings": 8,
  "duplicates": 67,
  "durationMs": 145320
}
```

### SCRAPE_FAILED

Triggered when a scrape job fails.

**Data structure:**
```json
{
  "jobId": "string",
  "error": "string",
  "siteName": "string"
}
```

### SYSTEM

General system notifications (account changes, announcements, etc.).

**Data structure:** Varies based on the specific system event.

## Real-time Updates

Notifications are also delivered in real-time via Socket.io. Connect to the WebSocket endpoint and listen for the `notification` event:

```javascript
import io from 'socket.io-client';

const socket = io('https://api.realtorspractice.com', {
  auth: {
    token: 'YOUR_TOKEN'
  }
});

socket.on('notification', (notification) => {
  console.log('New notification:', notification);
  // Update UI, show toast, increment badge count, etc.
});
```

```json
{
  "success": false,
  "error": "Authentication required",
  "message": "Unauthorized"
}
```

### 404 Not Found

```json
{
  "success": false,
  "error": "Notification not found or does not belong to this user",
  "message": "Notification not found"
}
```

```json
{
  "success": false,

  "message": "Failed to fetch notifications"
}
```
