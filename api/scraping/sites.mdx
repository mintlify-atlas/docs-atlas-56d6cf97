## Overview

Sites represent configured sources for scraping property listings. Each site contains CSS selectors, pagination rules, and metadata required for the scraper to extract data.

<Note>
Creating and updating sites requires **ADMIN** role. Viewing sites is available to all authenticated users.
</Note>

---

## List Sites

<CodeGroup>
```bash cURL
curl -X GET "https://api.realtorspractice.com/api/sites?page=1&limit=20&enabled=true" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://api.realtorspractice.com/api/sites?page=1&limit=20&enabled=true',
  {
    headers: {
      'Authorization': `Bearer ${token}`
    }
  }
);

const { data, pagination } = await response.json();
```

```python Python
response = requests.get(
    'https://api.realtorspractice.com/api/sites',
    headers={'Authorization': f'Bearer {token}'},
    params={'page': 1, 'limit': 20, 'enabled': True}
)

data = response.json()
```
</CodeGroup>

<ParamField query="page" type="number" default={1}>

## Create Site

<Note>
Requires **ADMIN** role.
</Note>

<CodeGroup>
```bash cURL
curl -X POST https://api.realtorspractice.com/api/sites \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "key": "buyletlive",
    "name": "BuyLetLive",
    "baseUrl": "https://buyletlive.com",
    "enabled": true,
    "listPaths": ["/properties/sale", "/properties/rent"],
    "selectors": {
      "listing_link": "a[href*=''/property/'']",
      "title": "h1.property-title",
      "price": "span.price, div.property-price",
      "location": "span.location, div.property-location",
      "bedrooms": "span.beds",
      "bathrooms": "span.baths",
      "images": "img.property-image::attr(src)",
      "description": "div.property-description",
      "features": "ul.amenities li"
    },
    "paginationType": "url_param",
    "maxPages": 5,
    "requiresBrowser": true
  }'
```

```javascript JavaScript
const response = await fetch('https://api.realtorspractice.com/api/sites', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    key: 'buyletlive',
    name: 'BuyLetLive',
    baseUrl: 'https://buyletlive.com',
    enabled: true,
    listPaths: ['/properties/sale', '/properties/rent'],
    selectors: {
      listing_link: "a[href*='/property/']",
      title: 'h1.property-title',
      price: 'span.price, div.property-price',
      location: 'span.location, div.property-location',
      bedrooms: 'span.beds',
      bathrooms: 'span.baths',
      images: 'img.property-image::attr(src)',
      description: 'div.property-description',
      features: 'ul.amenities li'
    },
    paginationType: 'url_param',
    maxPages: 5,
    requiresBrowser: true
  })
});

const { data } = await response.json();
```

```python Python

    'https://api.realtorspractice.com/api/sites',
    headers={
        'Authorization': f'Bearer {token}',
        'Content-Type': 'application/json'
    },
    json={
        'key': 'buyletlive',
        'name': 'BuyLetLive',
        'baseUrl': 'https://buyletlive.com',
        'enabled': True,
        'listPaths': ['/properties/sale', '/properties/rent'],
        'selectors': {
            'listing_link': "a[href*='/property/']",
            'title': 'h1.property-title',
            'price': 'span.price, div.property-price',
            'location': 'span.location, div.property-location',
            'bedrooms': 'span.beds',
            'bathrooms': 'span.baths',
            'images': 'img.property-image::attr(src)',
            'description': 'div.property-description',
            'features': 'ul.amenities li'
        },
        'paginationType': 'url_param',
        'maxPages': 5,
        'requiresBrowser': True
    }
)

data = response.json()['data']
```
</CodeGroup>

### Request Body

<ParamField body="key" type="string" required>
  Unique lowercase key (alphanumeric, hyphens, underscores only)
</ParamField>

<ParamField body="name" type="string" required>
  Display name (1-200 characters)
</ParamField>

<ParamField body="baseUrl" type="string" required>
  Base URL of the site (must be valid URL)
</ParamField>

<ParamField body="enabled" type="boolean" default={true}>
  Whether scraping is enabled for this site
</ParamField>

<ParamField body="parser" type="string" default="universal">
  Parser type to use
</ParamField>

<ParamField body="listPaths" type="string[]" default={[]}>
  Array of listing page paths to scrape (relative to baseUrl)
</ParamField>

<ParamField body="selectors" type="object">
  CSS selectors for extracting data from listing cards. Common fields:
  - `listing_link` - Link to property detail page (required)
  - `title` - Property title
  - `price` - Property price
  - `location` - Location/address
  - `bedrooms` - Number of bedrooms
  - `bathrooms` - Number of bathrooms
  - `toilets` - Number of toilets
  - `images` - Property images (use `::attr(src)` to extract src attribute)
  - `description` - Property description
  - `agentName` - Agent/seller name
  - `features` - List of features/amenities
  
  Use comma-separated selectors for fallbacks (e.g., `"h2.title, h1.property-title"`).
</ParamField>

<ParamField body="detailSelectors" type="object">
  CSS selectors for extracting data from property detail pages (optional, uses `selectors` if not provided)
</ParamField>

<ParamField body="paginationType" type="string" default="auto">
  Pagination strategy:
  - `auto` - Automatic detection
  - `next_button` - Click "Next" button
  - `page_number` - Click page numbers
  - `url_param` - Change URL query parameter (e.g., `?page=2`)
  - `path_segment` - Change URL path (e.g., `/page/2`)
  - `infinite_scroll` - Scroll to load more
  - `load_more` - Click "Load More" button
</ParamField>

<ParamField body="maxPages" type="number" default={30}>
  Maximum pages to scrape per path (1-200)
</ParamField>

<ParamField body="requiresBrowser" type="boolean" default={false}>
  Whether site requires JavaScript rendering. Set to `true` for SPAs or sites with dynamic content.
</ParamField>

<ParamField body="customHeaders" type="object">
  Custom HTTP headers to send with requests (e.g., User-Agent, Accept-Language)
</ParamField>

## Update Site

<Note>
Requires **ADMIN** role. Cannot change the `key` field.
</Note>

<CodeGroup>
```bash cURL
curl -X PUT https://api.realtorspractice.com/api/sites/clxy1111111111 \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "maxPages": 10,
    "enabled": true
  }'
```

```javascript JavaScript
const response = await fetch(
  'https://api.realtorspractice.com/api/sites/clxy1111111111',
  {
    method: 'PUT',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      maxPages: 10,
      enabled: true
    })
  }
);

const { data } = await response.json();
```

```python Python
response = requests.put(
    'https://api.realtorspractice.com/api/sites/clxy1111111111',
    headers={
        'Authorization': f'Bearer {token}',
        'Content-Type': 'application/json'
    },
    json={'maxPages': 10, 'enabled': True}
)

data = response.json()['data']
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="string" required>
  Site ID to update
</ParamField>

### Request Body

All fields from the create request are supported except `key`. All fields are optional (partial update).

### Response

Returns the updated site object.

## Selector Configuration Guide

### Basic Selectors

Use standard CSS selectors to target elements:

```json
{
  "title": "h1.property-title",
  "price": "span.price",
  "location": "address.location"
}
```

### Fallback Selectors

Use comma-separated selectors to provide fallbacks:

```json
{
  "title": "h2.listings-property-title, h1.property-title, .title",
  "price": "h3.listings-price, span.price, .property-price"
}
```

### Extracting Attributes

Use `::attr(attribute_name)` to extract attribute values:

```json
{
  "images": "img.property-image::attr(src), img.gallery-image::attr(data-src)",
  "listing_link": "a.property-link::attr(href)"
}
```

### Text Content Matching

Use `:contains()` for elements with specific text:

```json
{
  "bedrooms": "li:contains('Bedrooms'), span[title='Bedrooms']",
  "bathrooms": "li:contains('Bathrooms'), span.baths"
}
```

### List Extraction

For lists (features, amenities), use a selector that matches multiple elements:

```json
{
  "features": "li.feature, ul.amenities li, div.features span"
}
```

<Tip>
Test your selectors in the browser DevTools console using `document.querySelectorAll("your-selector")` before adding them to the site configuration.
</Tip>

## Site Health Monitoring

Each site tracks health metrics automatically:

- **healthScore** - Starts at 100, decreases by 20 per consecutive failure, resets to 100 on success
- **failCount** - Consecutive failures, resets to 0 on success
- **lastScrapeAt** - Timestamp of last scrape attempt
- **lastSuccessAt** - Timestamp of last successful scrape
- **avgListings** - Average listings found per successful scrape

<Warning>
Sites with `healthScore < 40` or `failCount >= 3` should be investigated for selector changes or site downtime.
</Warning>
