## Overview

Scrape jobs orchestrate the collection of property listings from configured sites. Jobs can be triggered manually or scheduled, and provide real-time progress updates via WebSocket.

### Job Types

- **PASSIVE_BULK** - Regular bulk scraping of all listings (lowest priority)
- **ACTIVE_INTENT** - User-triggered scrape based on search intent (highest priority)
- **RESCRAPE** - Re-scrape specific properties to update data
- **SCHEDULED** - Automated scheduled scraping

### Job Status

- **PENDING** - Job created, waiting to be dispatched
- **RUNNING** - Currently scraping
- **COMPLETED** - Finished successfully
- **FAILED** - Encountered critical error
- **CANCELLED** - Stopped by user or system

## Create Scrape Job

<Note>
Requires **ADMIN** or **EDITOR** role.
</Note>

<CodeGroup>
```bash cURL
curl -X POST https://api.realtorspractice.com/api/scrape/start \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "siteIds": ["clxx1234567890", "clxx0987654321"],
    "type": "ACTIVE_INTENT",
    "maxListingsPerSite": 50,
    "searchQuery": "3 bedroom apartments in Lekki",
    "parameters": {
      "minPrice": 50000000,
      "maxPrice": 150000000,
      "location": "Lekki"
    }
  }'
```

```javascript JavaScript
const response = await fetch('https://api.realtorspractice.com/api/scrape/start', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    siteIds: ['clxx1234567890', 'clxx0987654321'],
    type: 'ACTIVE_INTENT',
    maxListingsPerSite: 50,
    searchQuery: '3 bedroom apartments in Lekki',
    parameters: {
      minPrice: 50000000,
      maxPrice: 150000000,
      location: 'Lekki'
    }
  })
});

const { data } = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.realtorspractice.com/api/scrape/start',
    headers={
        'Authorization': f'Bearer {token}',
        'Content-Type': 'application/json'
    },
    json={
        'siteIds': ['clxx1234567890', 'clxx0987654321'],
        'type': 'ACTIVE_INTENT',
        'maxListingsPerSite': 50,
        'searchQuery': '3 bedroom apartments in Lekki',
        'parameters': {
            'minPrice': 50000000,
            'maxPrice': 150000000,
            'location': 'Lekki'
        }
    }
)

data = response.json()['data']
```
</CodeGroup>

## Real-Time Updates (WebSocket)

Subscribe to real-time job progress and logs via Socket.io.

### Connect to Scrape Namespace

```javascript WebSocket Client
import { io } from 'socket.io-client';

const socket = io('https://api.realtorspractice.com/scrape', {
  path: '/ws',
  auth: {
    token: 'YOUR_AUTH_TOKEN'
  }
});

// Join a specific job room
socket.emit('join:job', 'clxy9876543210');

// Listen for progress updates
socket.on('job:progress', (data) => {
  console.log('Progress:', data);
  // {
  //   jobId: 'clxy9876543210',
  //   processed: 25,
  //   total: 50,
  //   currentSite: 'PropertyPro',
  //   message: 'Processing page 3...',
  //   timestamp: '2026-03-11T14:33:00.000Z'
  // }
});

// Listen for log entries
socket.on('job:log', (data) => {
  console.log('Log:', data);
  // {
  //   jobId: 'clxy9876543210',
  //   level: 'info',
  //   message: 'Found 10 new listings on page 3',
  //   details: { page: 3, count: 10 },
  //   timestamp: '2026-03-11T14:33:05.000Z'
  // }
});

// Listen for completion
socket.on('job:completed', (data) => {
  console.log('Job completed:', data);
  // {
  //   jobId: 'clxy9876543210',
  //   stats: {
  //     totalListings: 47,
  //     newListings: 12,
  //     duplicates: 35,
  //     errors: 0
  //   },
  //   timestamp: '2026-03-11T14:38:45.000Z'
  // }
});

// Listen for errors
socket.on('job:error', (data) => {
  console.error('Job failed:', data);
  // {
  //   jobId: 'clxy9876543210',
  //   error: 'Connection timeout to PropertyPro',
  //   timestamp: '2026-03-11T14:35:00.000Z'
  // }
});

// Global job updates (all jobs)
socket.on('job_update', (data) => {
  console.log('Job status changed:', data);
  // { jobId: 'clxy9876543210', status: 'COMPLETED' }
});

// Leave job room when done
socket.emit('leave:job', 'clxy9876543210');
```

### Events

<ParamField body="job:progress" type="event">
  Progress update for a specific job (requires joining job room)
  
  **Payload:**
  - `jobId` - Job ID
  - `processed` - Items processed so far
  - `total` - Total items to process
  - `currentSite` - Site currently being scraped
  - `message` - Progress message
  - `timestamp` - ISO 8601 timestamp
</ParamField>

<ParamField body="job:log" type="event">
  New log entry for a specific job (requires joining job room)
  
  **Payload:**
  - `jobId` - Job ID
  - `level` - Log level (info | warning | error)
  - `message` - Log message
  - `details` - Additional structured data
  - `timestamp` - ISO 8601 timestamp
</ParamField>

<ParamField body="job:completed" type="event">
  Job completed successfully (requires joining job room)
  
  **Payload:**
  - `jobId` - Job ID
  - `stats` - Final statistics object
  - `timestamp` - ISO 8601 timestamp
</ParamField>

<ParamField body="job:error" type="event">
  Job failed with error (requires joining job room)
  
  **Payload:**
  - `jobId` - Job ID
  - `error` - Error message
  - `timestamp` - ISO 8601 timestamp
</ParamField>

<ParamField body="job_update" type="event">
  Global job status change notification (broadcast to all connected clients)
  
  **Payload:**
  - `jobId` - Job ID
  - `status` - New status (COMPLETED | FAILED | CANCELLED)
</ParamField>

<ParamField body="scrape_log" type="event">
  Global scrape log feed (broadcast to all connected clients)
  
  **Payload:**
  - Same as `job:log`
</ParamField>

## Job Priority

Jobs are dispatched to the scraper with different priorities:

| Type | Priority | Use Case |
|------|----------|----------|
| **ACTIVE_INTENT** | 1 (highest) | User-triggered searches requiring immediate results |
| **RESCRAPE** | 3 | Updating specific properties |
| **PASSIVE_BULK** | 7 | Background scraping of all listings |
| **SCHEDULED** | 7 | Automated periodic scraping |

<Tip>
Use `ACTIVE_INTENT` for user-facing searches to get results quickly. Use `PASSIVE_BULK` for overnight/background data collection.
</Tip>
