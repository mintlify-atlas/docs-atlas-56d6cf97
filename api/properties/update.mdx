## Endpoint

```
PUT /api/properties/:id
```

## Authentication

This endpoint requires authentication with **Editor** or **Admin** role.

**Headers:**
- `Authorization: Bearer <token>` - JWT token from Supabase authentication
- `Content-Type: application/json`

## Path Parameters

<ParamField path="id" type="string" required>
  Unique property identifier (CUID)
</ParamField>

## Request Body

All fields from the create endpoint are available for update, **except**:
- `listingUrl` (cannot be changed)
- `source` (cannot be changed)
- `siteId` (cannot be changed)

All fields are **optional** - only include the fields you want to update.

## Notes

- **Version Tracking**: Every update creates a new `PropertyVersion` record that stores:
  - Previous data state
  - New data state
  - List of changed fields
  - Change source (`MANUAL_EDIT`)
  - User who made the change
  - Timestamp
- **Quality Score Recalculation**: The quality score is automatically recalculated based on the merged data (existing + updates).
- **Immutable Fields**: The fields `listingUrl`, `source`, and `siteId` cannot be changed after creation.
- **Version Increment**: The `currentVersion` field is automatically incremented with each update.
- **Search Index Update**: The property is automatically re-indexed in Meilisearch after update.
- **Partial Updates**: You only need to include the fields you want to update - all other fields remain unchanged.
- **Price History**: Price changes are NOT automatically tracked through this endpoint. Use the dedicated price history endpoint if you need to track price changes over time.
