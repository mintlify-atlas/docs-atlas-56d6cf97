## Endpoint

```
DELETE /api/properties/:id
```

## Authentication

This endpoint requires authentication with **Admin** role only.

**Headers:**
- `Authorization: Bearer <token>` - JWT token from Supabase authentication

## Path Parameters

<ParamField path="id" type="string" required>
  Unique property identifier (CUID)
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="null">
  Null value
</ResponseField>

<ResponseField name="message" type="string">
  Success message: "Property deleted"
</ResponseField>

## Error Responses

<ResponseField name="401 Unauthorized">
  Authentication token is missing or invalid
  
  ```json
  {
    "success": false,
    "error": "Unauthorized",
    "message": "Authentication required"
  }
  ```
</ResponseField>

<ResponseField name="403 Forbidden">
  User does not have Admin role
  
  ```json
  {
    "success": false,
    "error": "Forbidden",
    "message": "Admin role required"
  }
  ```
</ResponseField>

<ResponseField name="404 Not Found">
  Property with the specified ID does not exist or has already been deleted
  
  ```json
  {
    "success": false,
    "error": "Property not found",
    "message": "Property not found"
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Server error while deleting property
  
  ```json
  {
    "success": false,
    "error": "Failed to delete property",
    "message": "Failed to delete property"
  }
  ```
</ResponseField>

```javascript TypeScript
const propertyId = 'clx1a2b3c4d5e6f7g8h9i0';
```

## Notes

- **Soft Delete**: This operation performs a soft delete by setting the `deletedAt` timestamp. The property record is not permanently removed from the database.
- **Search Index Removal**: The property is automatically removed from the Meilisearch index after deletion.
- **Cascading Effects**: Related records (versions, price history, saved search matches) are NOT deleted, but they will be inaccessible through standard queries due to cascade rules.
- **Admin Only**: Only users with the Admin role can delete properties. Editors and Viewers cannot perform this action.
- **Idempotency**: Attempting to delete an already-deleted property will return a 404 error.
- **Restoration**: Deleted properties can be restored using the bulk action endpoint with action `restore`.

## Bulk Delete

For deleting multiple properties at once, you can implement a bulk delete operation:

```javascript
const response = await fetch(
  'https://realtors-practice-new-api.onrender.com/api/properties/bulk-action',
  {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      ids: ['clx1a2b3c4d5e6f7g8h9i0', 'clx9z8y7x6w5v4u3t2s1'],
      action: 'delete'
    })
  }
);
```

## Related Endpoints

- **Restore Property**: Use `POST /api/properties/bulk-action` with `action: "restore"`
- **Permanently Delete**: There is no endpoint for permanent deletion. Contact system administrators for hard deletes.

## Use Cases

### Delete a single property
```javascript
await fetch(
  `https://realtors-practice-new-api.onrender.com/api/properties/${propertyId}`,
  {
    method: 'DELETE',
    headers: { 'Authorization': `Bearer ${token}` }
  }
);
```

### Delete with confirmation
```javascript
const confirmDelete = confirm('Are you sure you want to delete this property?');

if (confirmDelete) {
  const response = await fetch(
    `https://realtors-practice-new-api.onrender.com/api/properties/${propertyId}`,
    {
      method: 'DELETE',
      headers: { 'Authorization': `Bearer ${token}` }
    }
  );
  
  if (response.ok) {
    alert('Property deleted successfully');
    // Redirect or update UI
  }
}
```

### Delete with error handling
```javascript
try {
  const response = await fetch(
    `https://realtors-practice-new-api.onrender.com/api/properties/${propertyId}`,
    {
      method: 'DELETE',
      headers: { 'Authorization': `Bearer ${token}` }
    }
  );
  
  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message);
  }
  
  console.log('Property deleted successfully');
} catch (error) {
  console.error('Delete failed:', error.message);
  // Show error to user
}
```
