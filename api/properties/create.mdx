## Endpoint

```
POST /api/properties
```

## Authentication

This endpoint requires authentication with **Editor** or **Admin** role.

**Headers:**
- `Authorization: Bearer <token>` - JWT token from Supabase authentication
- `Content-Type: application/json`

## Request Body

  Indicates if the request was successful
</ResponseField>

## Error Responses

<ResponseField name="401 Unauthorized">
  Authentication token is missing or invalid
  
  ```json
  {
    "success": false,
    "error": "Unauthorized",
    "message": "Authentication required"
  }
  ```
</ResponseField>

<ResponseField name="403 Forbidden">
  User does not have Editor or Admin role
  
  ```json
  {
    "success": false,
    "error": "Forbidden",
    "message": "Admin or Editor role required"
  }
  ```
</ResponseField>

<ResponseField name="400 Bad Request">
  Validation error with request body
  
  ```json
  {
    "success": false,
    "error": "Validation error",
    "message": "Invalid request body"
  }
  ```
</ResponseField>

<ResponseField name="409 Conflict">
  Duplicate property detected (same title, URL, and source)
  
  ```json
  {
    "success": false,
    "error": "Duplicate property",
    "message": "Existing property: clx1a2b3c4d5e6f7g8h9i0"
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Server error while creating property
  
  ```json
  {
    "success": false,
    "error": "Failed to create property",
    "message": "Failed to create property"
  }
  ```
</ResponseField>

## Notes

- **Duplicate Detection**: The system automatically generates a hash based on `title`, `listingUrl`, and `source` to detect duplicates. If a duplicate is found, a 409 error is returned with the existing property ID.
- **Quality Score**: A quality score (0-100) is automatically calculated based on data completeness and quality.
- **Price History**: If a `price` is provided, an initial price history record is automatically created.
- **Change Source**: All manual creations are tracked with `changeSource: "MANUAL_EDIT"` and linked to the authenticated user.
- **Search Indexing**: The property is automatically indexed in Meilisearch for full-text search.
- **Version Tracking**: The property is created with `currentVersion: 1`.
- **Default Values**: Fields like `status`, `verificationStatus`, `category`, `priceCurrency`, `state`, and `country` have default values if not provided.
