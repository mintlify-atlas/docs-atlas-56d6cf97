---
title: "Get Property"
description: "Retrieve detailed information about a single property by ID"
---

## Endpoint

```
GET /api/properties/:id
```

## Authentication

This endpoint requires authentication. All authenticated users (Viewer, Editor, Admin) can access this endpoint.

**Headers:**
- `Authorization: Bearer <token>` - JWT token from Supabase authentication

## Path Parameters

<ParamField path="id" type="string" required>
  Unique property identifier (CUID)
</ParamField>

## Error Responses

<ResponseField name="401 Unauthorized">
  Authentication token is missing or invalid
  
  ```json
  {
    "success": false,
    "error": "Unauthorized",
    "message": "Authentication required"
  }
  ```
</ResponseField>

<ResponseField name="404 Not Found">
  Property with the specified ID does not exist or has been deleted
  
  ```json
  {
    "success": false,
    "error": "Property not found",
    "message": "Property not found"
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Server error while fetching property
  
  ```json
  {
    "success": false,
    "error": "Failed to fetch property",
    "message": "Failed to fetch property"
  }
  ```
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl -X GET "https://realtors-practice-new-api.onrender.com/api/properties/clx1a2b3c4d5e6f7g8h9i0" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const propertyId = 'clx1a2b3c4d5e6f7g8h9i0';

const response = await fetch(
  `https://realtors-practice-new-api.onrender.com/api/properties/${propertyId}`,
  {
    headers: {
      'Authorization': `Bearer ${token}`
    }
  }
);

const { data: property } = await response.json();
console.log(property.title);
console.log(property.price);
console.log(property.bedrooms);
```

## Notes

- The response includes additional fields compared to the list endpoint:
  - `site.baseUrl` - Full base URL of the source site
  - `_count` - Counts of related records (versions and price history)
- Only non-deleted properties can be retrieved (where `deletedAt` is null)
- The `qualityScore` is automatically calculated based on data completeness and quality
- Version history and price history can be accessed through separate endpoints (see `/api/properties/:id/versions` and `/api/properties/:id/price-history`)
