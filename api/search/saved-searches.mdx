## Overview

Saved searches allow authenticated users to save filter criteria and receive notifications when new properties match their criteria. The system automatically checks all active saved searches periodically and creates match records.

<Note>
  All saved search endpoints require authentication via `Authorization: Bearer <token>` header.
</Note>

---

## Create Saved Search

### POST /api/saved-searches

Create a new saved search with filter criteria and notification preferences.

#### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token from Supabase authentication
</ParamField>

<ParamField header="Content-Type" type="string" default="application/json">
  Must be `application/json`
</ParamField>

#### Request Body

<ParamField body="name" type="string" required>
  Display name for the saved search (1-100 characters)
</ParamField>

<ParamField body="description" type="string">
  Optional description (max 500 characters)
</ParamField>

<ParamField body="filters" type="object" required>
  Search filter criteria
  
  <ParamField body="listingType" type="string">
    SALE, RENT, LEASE, or SHORTLET
  </ParamField>
  
  <ParamField body="category" type="string">
    RESIDENTIAL, COMMERCIAL, LAND, SHORTLET, INDUSTRIAL
  </ParamField>
  
  <ParamField body="minPrice" type="number">
    Minimum price in NGN
  </ParamField>
  
  <ParamField body="maxPrice" type="number">
    Maximum price in NGN
  </ParamField>
  
  <ParamField body="bedrooms" type="integer">
    Number of bedrooms (exact match)
  </ParamField>
  
  <ParamField body="state" type="string">
    Nigerian state (e.g., "Lagos", "Abuja")
  </ParamField>
  
  <ParamField body="area" type="string">
    Specific area/neighborhood (case-insensitive partial match)
  </ParamField>
  
  <ParamField body="propertyType" type="string">
    Property type (e.g., "Flat", "Duplex", "Detached")
  </ParamField>
  
  <ParamField body="features" type="array">
    Array of required features (e.g., ["Swimming Pool", "Gym"])
  </ParamField>
</ParamField>

<ParamField body="naturalQuery" type="string">
  Optional natural language query string for reference
</ParamField>

<ParamField body="notifyEmail" type="boolean" default="false">
  Send email notifications for new matches
</ParamField>

<ParamField body="notifyInApp" type="boolean" default="true">
  Create in-app notifications for new matches
</ParamField>

#### Response

<ResponseField name="success" type="boolean" required>
  Whether the request succeeded
</ResponseField>

<ResponseField name="data" type="object" required>
  Created saved search object
  
  <ResponseField name="id" type="string">
    Saved search CUID
  </ResponseField>
  
  <ResponseField name="userId" type="string">
    Owner user ID
  </ResponseField>
  
  <ResponseField name="name" type="string">
    Search name
  </ResponseField>
  
  <ResponseField name="description" type="string">
    Search description
  </ResponseField>
  
  <ResponseField name="filters" type="object">
    Filter criteria object
  </ResponseField>
  
  <ResponseField name="isActive" type="boolean">
    Whether the search is actively checking for matches
  </ResponseField>
  
  <ResponseField name="matchCount" type="integer">
    Total number of matches found
  </ResponseField>
  
  <ResponseField name="newSinceCheck" type="integer">
    New matches since last check
  </ResponseField>
  
  <ResponseField name="notifyEmail" type="boolean">
    Email notification preference
  </ResponseField>
  
  <ResponseField name="notifyInApp" type="boolean">
    In-app notification preference
  </ResponseField>
  
  <ResponseField name="createdAt" type="string">

  </ResponseField>

## Update Saved Search

### PUT /api/saved-searches/:id

Update an existing saved search. Only the owner can update their saved searches.

#### Parameters

<ParamField path="id" type="string" required>
  Saved search CUID
</ParamField>

#### Request Body

All fields are optional. Only provided fields will be updated.

<ParamField body="name" type="string">
  Updated search name
</ParamField>

<ParamField body="description" type="string">
  Updated description
</ParamField>

<ParamField body="filters" type="object">
  Updated filter criteria (replaces entire filters object)
</ParamField>

<ParamField body="isActive" type="boolean">
  Enable or disable automatic match checking
</ParamField>

<ParamField body="notifyEmail" type="boolean">
  Update email notification preference
</ParamField>

<ParamField body="notifyInApp" type="boolean">
  Update in-app notification preference
</ParamField>

#### Example Request

<CodeGroup>
```bash cURL - Pause Search

  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Content-Type: application/json" \
```

## Background Match Checking

The system runs a cron job (via `SavedSearchService.checkAllActive()`) that:

1. Queries all active saved searches
2. For each search, queries the database for properties matching the filter criteria
3. Only considers properties created/updated since `lastCheckedAt`
4. Creates `SavedSearchMatch` records for new matches (skips duplicates)
5. Updates `matchCount` and `newSinceCheck` counters
6. Triggers notifications based on `notifyEmail` and `notifyInApp` preferences

**Match checking logic:**
- Exact match on: `listingType`, `category`, `state`, `bedrooms`
- Range match on: `minPrice` / `maxPrice`
- Case-insensitive partial match on: `area`, `propertyType`
- Array contains on: `features` (all specified features must be present)

### Notification Types

- **In-app:** Creates `Notification` record with type `NEW_MATCH`
- **Email:** Sends email digest of new matches (implementation pending)

## Notes

- Maximum 100 matches returned per `checkMatches()` run to prevent runaway queries
- Saved searches are soft-deleted when user account is deleted (cascade)
- All match records are deleted when a saved search is deleted
- `lastCheckedAt` timestamp is updated even when no new matches are found
- The `newSinceCheck` counter resets on each check cycle
