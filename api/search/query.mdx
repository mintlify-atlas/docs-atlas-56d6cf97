## POST /api/search

Execute a full-text search query with faceted filtering, natural language parsing, and automatic scrape triggering on zero results.

### Headers

<ParamField header="Content-Type" type="string" default="application/json">
  Must be `application/json`
</ParamField>

### Query Parameters

<ParamField query="q" type="string">
  Search query string. Supports natural language parsing:
  - Price ranges: "under 5m", "less than 2m", "over 10m"
  - Bedrooms: "3 bed", "4 bedroom", "2 bhk"
  - Listing type: "rent", "sale", "shortlet"
  - Location: "lekki", "victoria island", "ajah"
  
  Example: `"3 bed flat for rent in lekki under 2m"`
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of results to return (max 100)
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Pagination offset for result set
</ParamField>

<ParamField query="facets" type="string">
  Comma-separated list of fields to return facet distributions for.
  
  **Default facets:** `categoryName,listingType,bedrooms,state`
  
  **Available facets:**
  - `categoryName` - Property category (RESIDENTIAL, COMMERCIAL, LAND, etc.)
  - `listingType` - SALE, RENT, LEASE, SHORTLET
  - `bedrooms` - Number of bedrooms
  - `bathrooms` - Number of bathrooms
  - `state` - Nigerian state (Lagos, Abuja, etc.)
  - `area` - Specific area/neighborhood
  - `furnishing` - FURNISHED, SEMI_FURNISHED, UNFURNISHED
  - `propertyType` - Flat, Duplex, Detached, etc.
</ParamField>

<ParamField query="filters" type="string">
  Meilisearch filter expression. Multiple conditions combined with `AND` or `OR`.
  
  **Filter syntax:**
  - Equality: `listingType = RENT`
  - Range: `price >= 1000000 AND price <= 5000000`
  - Array contains: `features = "Swimming Pool"`
  - Nested: `(bedrooms = 3 OR bedrooms = 4) AND listingType = RENT`
  
  **Filterable fields:**
  - `listingType` (string enum)
  - `categoryName` (string enum)
  - `price` (number)
  - `bedrooms` (integer)
  - `bathrooms` (integer)
  - `state` (string)
  - `area` (string)
  - `lga` (string - Local Government Area)
  - `furnishing` (string enum)
  - `propertyType` (string)
  - `features` (array of strings)
</ParamField>

<ParamField query="sort" type="string">
  Sort order. Multiple sorts separated by comma.
  
  **Default:** `createdAt:desc`
  
  **Available sorts:**
  - `price:asc` / `price:desc`
  - `createdAt:asc` / `createdAt:desc`
  - `bedrooms:asc` / `bedrooms:desc`
  - `qualityScore:desc`
  
  Example: `price:asc,createdAt:desc`
</ParamField>

### Example Requests

<CodeGroup>
```bash cURL - Natural Language
curl -X GET "https://realtors-practice-new-api.onrender.com/api/search?q=3+bedroom+flat+for+rent+in+lekki+under+2m&limit=10" \
  -H "Content-Type: application/json"
```

```bash cURL - Explicit Filters
curl -X GET "https://realtors-practice-new-api.onrender.com/api/search?filters=bedrooms+%3D+3+AND+listingType+%3D+RENT+AND+state+%3D+Lagos+AND+price+%3C%3D+2000000&sort=price:asc&limit=20" \
  -H "Content-Type: application/json"
```

```javascript JavaScript - Natural Language
const response = await fetch(
  'https://realtors-practice-new-api.onrender.com/api/search?' + new URLSearchParams({
    q: '3 bedroom flat for rent in lekki under 2m',
    limit: '10'
  })
);

const data = await response.json();
console.log(`Found ${data.data.estimatedTotalHits} properties`);
```

```javascript JavaScript - Advanced Filtering
const response = await fetch(
  'https://realtors-practice-new-api.onrender.com/api/search?' + new URLSearchParams({
    filters: '(bedrooms = 3 OR bedrooms = 4) AND listingType = RENT AND price >= 1000000 AND price <= 3000000',
    facets: 'area,bedrooms,furnishing',
    sort: 'price:asc',
    limit: '20',
    offset: '0'
  })
);

const data = await response.json();

// Use facets for filter UI
const areaFacets = data.data.facetDistribution.area;
console.log('Available areas:', Object.keys(areaFacets));
```
</CodeGroup>

### Filter Examples

#### Price Range
```
price >= 500000 AND price <= 3000000
```

#### Multiple Bedrooms
```
(bedrooms = 2 OR bedrooms = 3 OR bedrooms = 4)
```

#### Location + Type + Price
```
state = Lagos AND area = "Lekki Phase 1" AND listingType = RENT AND price <= 2500000
```

#### Feature-based Search
```
features = "Swimming Pool" AND features = "Gym" AND bedrooms >= 3
```

#### Commercial Properties
```
categoryName = COMMERCIAL AND state = Lagos AND price >= 10000000
```

### Natural Language Parsing

The search endpoint automatically extracts structured filters from natural language queries:

| Input Query | Extracted Filters |
|-------------|------------------|
| `"3 bed flat for rent"` | `bedrooms = 3`, `listingType = RENT` |
| `"under 5m in lekki"` | `price <= 5000000`, location text search for "lekki" |
| `"4 bedroom house over 10m"` | `bedrooms = 4`, `price >= 10000000` |
| `"shortlet in vi"` | `listingType = SHORTLET`, location text search for "vi" |

### Zero Results Behavior

When a search returns zero results and the query is valid (length > 2 characters), the system automatically triggers an **active scrape** job targeting PropertyPro and Nigeria Property Centre with the search parameters. This ensures the database grows to match user intent.

### Rate Limiting

- **Public search:** 200 requests per 5 minutes per IP
- Returns `429 Too Many Requests` when limit exceeded
- Message: "Too many search requests, please try again later."

### Error Responses

<CodeGroup>
```json 500 - Meilisearch Unavailable

  "data": {
    "hits": [],
    "nbHits": 0,

```

```json 429 - Rate Limited

  "success": false,
  "error": "Too many search requests, please try again later."
}
```
</CodeGroup>

### Notes

- Meilisearch indexes are updated in real-time when properties are scraped or edited
- Search queries are typo-tolerant (Meilisearch default: 2 typos allowed)
- Results are ranked by relevance score, then by quality score
- All properties with `deletedAt != null` are excluded from search
- Maximum of 100 results per request (use pagination for more)
